# 文章详情 - 前端接口文档

基础路径: `/api/v1`

通用响应格式:
```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

## 1. 获取文章详情

**GET** `/posts/{slug}`

根据文章 `slug` 获取完整文章信息（含正文内容）。

### 请求参数

| 参数 | 类型 | 必填 | 说明 |

| slug | string | 是 | 文章URL标识（路径参数） |


### 响应字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| id | long | 文章ID |
| title | string | 文章标题 |
| slug | string | URL标识 |
| excerpt | string | 摘要 |
| featuredImage | string | 特色图片URL |
| status | int | 0-草稿, 1-已发布, 2-私密 |
| authorId | long | 作者ID |
| categoryId | long | 分类ID |
| publishedAt | string | 发布时间 |
| createdAt | string | 创建时间 |
| updatedAt | string | 更新时间 |
| viewCount | int | 浏览量 |
| likeCount | int | 点赞数 |
| commentCount | int | 评论数 |
| commentEnable | int | 0-不允许评论, 1-允许评论 |
| author | object | { id, nickname, avatar } |
| category | object | { id, name, slug } |
| tags | array | [{ id, name, slug }] |
| body | string | 正文（Markdown格式） |
| prevPost | object | 上一篇：{ id, title, slug }，没有则为 null |
| nextPost | object | 下一篇：{ id, title, slug }，没有则为 null |

### 上下篇说明

- `prevPost` / `nextPost` 在同专栏内按 `series_posts.sort_order` 升序排列
- 文章没有所属专栏时，`prevPost` 和 `nextPost` 均为 `null`
- 专栏内第一篇没有 `prevPost`，最后一篇没有 `nextPost`

### 异常情况

- `slug` 不存在 → `code: 404, message: "文章不存在"`

## 2. 浏览上报

**POST** `/posts/{id}/view`

访问文章详情页时上报一次浏览，浏览量 +1。

### 请求参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | long | 是 | 文章ID（路径参数） |

### 响应示例

```json
{
  "code": 0,
  "message": "success",
  "data": null
}
```

### 防刷策略

前端通过 `sessionStorage` 控制同一标签页内只上报一次：

```
用户打开文章详情页
  → sessionStorage 是否有 viewed_{postId}
  → 没有 → 调用 POST /view → view_count +1 → 写入 sessionStorage
  → 有 → 不再上报
  ↓
关闭标签页 → sessionStorage 清除
下次打开 → 重新计一次
```

## 3. 点赞

**POST** `/posts/{id}/like`

为文章点赞。

### 请求参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | long | 是 | 文章ID（路径参数） |
| X-Visitor-Id | string | 是 | 浏览器指纹（请求头） |

### 响应示例

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "liked": true,
    "likeCount": 90
  }
}
```

### 说明

- 幂等：同一 `visitor_id` 对同一篇文章重复点赞返回 `liked: true`，不重复计数
- 点赞后 `posts.like_count` 同步 +1

### 异常情况

- 文章不存在 → `code: 404, message: "文章不存在"`
- `X-Visitor-Id` 为空 → `code: 400, message: "visitorId不能为空"`

## 4. 取消点赞

**DELETE** `/posts/{id}/like`

取消对文章的点赞。

### 请求参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | long | 是 | 文章ID（路径参数） |
| X-Visitor-Id | string | 是 | 浏览器指纹（请求头） |

### 响应示例

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "liked": false,
    "likeCount": 89
  }
}
```

### 说明

- 幂等：未点赞状态下取消返回 `liked: false`，不重复扣减
- 取消后 `posts.like_count` 同步 -1

### 异常情况

- `X-Visitor-Id` 为空 → `code: 400, message: "visitorId不能为空"`

## 5. 查询点赞状态

**GET** `/posts/{id}/like/status`

查询当前访客是否已点赞。

### 请求参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | long | 是 | 文章ID（路径参数） |
| X-Visitor-Id | string | 是 | 浏览器指纹（请求头） |

### 响应示例

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "liked": true
  }
}
```

### 说明

- 不校验文章是否存在，不存在返回 `liked: false`

### 异常情况

- `X-Visitor-Id` 为空 → `code: 400, message: "visitorId不能为空"`

## 6. 数据库

### like_records 表（点赞记录）

```sql
CREATE TABLE `like_records` (
  `id` bigint NOT NULL AUTO_INCREMENT,
  `post_id` bigint NOT NULL COMMENT '文章ID',
  `visitor_id` varchar(64) NOT NULL COMMENT '浏览器指纹',
  `created_at` datetime NOT NULL DEFAULT CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_post_visitor` (`post_id`, `visitor_id`),
  KEY `idx_post_id` (`post_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_0900_ai_ci COMMENT='点赞记录表';
```

### 说明

- `uk_post_visitor` 联合唯一索引保证同一人对同一篇文章只能点赞一次
- 取消点赞直接删除记录，不做软删除
