# 接口测试计划

> 依据: 1安全模块设计方案 + 实际控制器代码
> 范围: 4 个 Controller，约 25+ 个接口
> 环境: `POST http://localhost:8099/iblogserver/api/v1`

---

## 一、测试综述

### 测试策略

| 维度 | 说明 |
|------|------|
| 功能测试 | 每个接口的正向流程 + 主流异常场景 |
| 安全测试 | 未认证访问、权限不足、Token 篡改/过期/吊销 |
| 边界测试 | 分页边界、参数边界、特殊字符 |
| 并发测试 | Refresh Token Rotation 竞争 |
| 幂等测试 | 重复点赞/取消点赞、重复删除 |

### 通用前提

- 数据库需预置：至少 1 个管理员用户、5 篇以上文章（覆盖草稿/发布/私密）、若干分类/标签/专栏
- 图片测试：准备一张 1MB 内的 jpg 和一张超过 5MB 的图片
- Token 缓存：测试过程中需记录每个场景的 access_token 和 refresh_token

## 二、AuthController（认证模块）

### 2.1 POST /auth/login — 登录

| 编号 | 场景 | 输入 | 预期结果 |
|------|------|------|----------|
| A-01 | **正向登录** | 正确 username + password | code=0, 返回 accessToken, refreshToken, expiresIn, user 信息 |
| A-02 | **错误密码** | 正确 username + 错误 password | code=401, "用户名或密码错误" |
| A-03 | **不存在用户** | 不存在的 username | code=401 |
| A-04 | **空请求体** | `{}` | code=401 |
| A-05 | **缺失字段** | 只传 username 不传 password | code=401 |
| A-06 | **DelegatingPasswordEncoder 兜底** | 旧明文密码用户 | 登录成功，下次登录应可使用 bcrypt |
| A-07 | **被禁用用户** | status=disabled 的用户 | 登录失败（需确认具体错误码） |
| A-08 | **特殊字符用户名** | 含 `<>"'&` 的用户名 | 无 SQL 注入，返回正确错误 |
| A-09 | **凭证类型校验** | credentials 中存原始 token 而非 hash | 登出时能从 credentials 提取原始 token |

| 编号 | 场景 | 输入 | 预期结果 |

### 2.4 POST /auth/refresh — 刷新令牌

| 编号 | 场景 | 输入 | 预期结果 |
|------|------|------|----------|
| A-21 | **正向刷新** | 有效 refresh_token | code=0, 返回新的 TokenPair（access + refresh 均更新） |
| A-22 | **旧 Refresh Token 失效验证** | 刷新后使用旧 refresh_token 再次刷新 | code=401, 旧 token 已 revoked=1 |
| A-23 | **多 tab 并发刷新（Rotation 兜底）** | 同一 refresh_token 并发 3 次请求 | 至少 2 个请求成功（replaced_by 链式查找），无数据损坏 |
| A-24 | **空 refresh_token** | `{"refreshToken": ""}` | code=400, "refreshToken 不能为空" |
| A-25 | **无效 refresh_token** | `{"refreshToken": "garbage"}` | code=401 |
| A-26 | **过期 refresh_token** | 7 天前签发的 token | code=401 |
| A-27 | **使用 Access Token 刷新** | 将 access_token 当 refresh_token 传入 | code=401（type 校验拒绝） |
| A-28 | **已吊销的 refresh_token** | 已被 revoked 且无 replaced_by 链 | code=401 |

### 测试用 refresh_token 时效
由于 access_token 有效期 2h、refresh_token 7d，本地手动测试过期场景需要：
- **方式一**：修改 `application.yml` 临时将过期时间设短（如 access 1min, refresh 5min）
- **方式二**：直接查询 `user_tokens` 表，将 `expires_at` 手动改为过去时间
- **方式三**：调用登出接口，使 token revoked=1 来模拟吊销
