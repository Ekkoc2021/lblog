# 安全模块设计方案

> 基于 Spring Security + 双自定义 Token（Access + Refresh）的自建鉴权系统。
> 设计时考虑了后续将 Token 存储迁移至 Redis 的扩展路径。

---

## 一、整体架构

```
┌─────────────────────────────────────────────────────────┐
│                   客户端 (React)                          │
│   Header: Authorization: Bearer <access_token>           │
│   (refresh_token 存储在内存/本地存储，由前端管理)           │
└────────────────────┬────────────────────────────────────┘
                     │
┌────────────────────▼────────────────────────────────────┐
│              Spring Security 过滤器链                      │
│                                                          │
│  ├─ LogoutFilter                                         │
│  ├─ TokenAuthenticationFilter (自定义)                    │
│  │   └─ 从 Authorization header 提取 access_token        │
│  │   └─ TokenService.validateAccessToken()               │
│  │   └─ 成功后 → SecurityContextHolder.setAuthentication │
│  ├─ UsernamePasswordAuthenticationFilter                 │
│  │   └─ /login 走 AuthenticationManager                  │
│  │   └─ LoginSuccessHandler → 签发双 token               │
│  └─ FilterSecurityInterceptor                           │
└────────────────────┬────────────────────────────────────┘
                     │
┌────────────────────▼────────────────────────────────────┐
│               TokenService (业务门面)                     │
│                                                          │
│  ┌──────────────────────────────────────────┐            │
│  │ TokenRepository (存储抽象层)              │            │
│  │  ├─ DbTokenRepository (当前实现 — MySQL) │            │
│  │  └─ RedisTokenRepository (未来实现)      │            │
│  └──────────────────────────────────────────┘            │
└──────────────────────────────────────────────────────────┘
```

### 关键设计原则

1. **TokenRepository 接口抽象** — TokenService 只依赖接口，存储层可切换
2. **服务端不存原始 token** — 数据库/Redis 只存 SHA-256 哈希
3. **Refresh Token Rotation** — 每次刷新时旧 refresh token 立即失效，签发新的
4. **无状态认证 + 可查询状态** — Token 本身无状态（不编码用户数据），但支持服务端撤销

## 二、Token 设计

### 2.1 Token 格式

```
原始 Token: 32 字节 SecureRandom → Base64 URL-safe 编码 (43 字符)
示例: xY7vE3kP9qR2mL8nA4bC5dF6gH1jK0sS2tU3vW4xZ5

服务端存储: SHA-256(raw_token) → 64 字符十六进制字符串
```

### 2.2 Token 类型

| 类型 | 有效期 | 传输方式 | 用途 |
|------|--------|----------|------|
| Access Token | 2 小时 | Authorization header | 访问受保护资源 |
| Refresh Token | 7 天 | JSON body 传递 | 静默续期 access token |

### 2.3 Token 生命周期

```
登录成功
  └→ 签发 TokenPair { access_token, refresh_token }
      └→ access_token 存入 SecurityContext 当前请求
      └→ 客户端保存双 token

访问受保护接口
  └→ 携带 access_token → TokenAuthenticationFilter 验证
  └→ 验证流程: 提取 token → SHA-256 → 查 TokenRepository → 检查 revoked & expires_at

Access Token 过期
  └→ 客户端收到 401 → 调用 POST /auth/refresh
  └→ 服务端验证 refresh_token (校验 + rotation)
  └→ 旧 refresh_token → revoked=1
  └→ 签发新的 TokenPair

主动登出
  └→ POST /auth/logout
  └→ 从 SecurityContext 取当前 access_token → revoked=1
  └→ 可选择同时吊销该用户所有 refresh_token
```

## 三、TokenRepository 接口设计

这是整个安全模块的存储抽象层，后续迁移到 Redis 只需新增一个实现类。

```java
public interface TokenRepository {

    /** 存储一个 token 记录 */
    void save(TokenRecord record);

    /** 根据 tokenHash 查找有效的 token 记录 */
    Optional<TokenRecord> findByHash(String tokenHash);

    /** 吊销单个 token */
    void revoke(String tokenHash);

    /** 吊销用户所有 token */
    void revokeAllByUserId(Long userId);

    /** 吊销用户所有指定类型的 token */
    void revokeAllByUserId(Long userId, String tokenType);

    /** 清理过期 token */
    int deleteExpired();

    /** 统计用户有效 token 数 */
    int countValidByUserId(Long userId);
}
```

```java
public class TokenRecord {
    private Long id;
    private Long userId;
    private String tokenHash;   // SHA-256
    private String tokenType;   // ACCESS / REFRESH
    private LocalDateTime expiresAt;
    private boolean revoked;
    private String replacedBy;  // rotation 时被哪个新 token 替换
}
```

### 存储对比

| 能力 | DbTokenRepository (MySQL) | RedisTokenRepository (未来) |
|------|--------------------------|-----------------------------|
| 查询 | SQL 索引 | O(1) 直接读取 |
| 过期清理 | 定时任务 DELETE | TTL 自动过期 |
| 批量吊销 | UPDATE 语句 | 用 Set 结构 + DEL |
| 复杂度 | 低 | 中（需处理 TTL + 原子性） |

未来迁移时，TokenService 业务逻辑不变，只需：
1. 实现 `RedisTokenRepository implements TokenRepository`
2. 配置类中 `@Bean @ConditionalOnProperty` 切换实现

|------|------|------|------|
| `POST /api/v1/auth/login` | 公开 | — | 登录 |
| `POST /api/v1/auth/refresh` | 无 Bearer 认证 | 需 body 中 refresh_token 校验通过 | 续期 |
| `GET /api/v1/auth/me` | 认证 | 任意登录用户 | 当前用户信息 |
| `POST /api/v1/auth/logout` | 认证 | 任意登录用户 | 登出 |
| `POST /api/v1/auth/change-password` | 认证 | 任意登录用户 | 修改密码 |
| `GET /api/v1/home/**` | 公开 | — | 博客前台 |
| `GET /uploads/**` | 公开 | — | 静态文件 |
| `/api/v1/admin/**` | 认证 + role=admin | 管理员 | 后台管理 |
| `/error` | 公开 | — | Spring Boot 错误转发 |

### SecurityConfig 代码结构示意

```java
@Configuration
@EnableWebSecurity
@EnableMethodSecurity
public class SecurityConfig {

    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http
            .csrf(AbstractHttpConfigurer::disable)
            .cors(cors -> cors.configurationSource(corsConfigurationSource()))
            .session(session -> session.sessionCreationPolicy(STATELESS))
            .exceptionHandling(ex -> ex
                .authenticationEntryPoint(customAuthEntryPoint)
                .accessDeniedHandler(customAccessDeniedHandler))
            .authorizeHttpRequests(auth -> auth
                .requestMatchers(POST, "/api/v1/auth/login").permitAll()
                .requestMatchers(POST, "/api/v1/auth/refresh").permitAll()
                .requestMatchers(GET, "/api/v1/home/**").permitAll()
                .requestMatchers(GET, "/uploads/**").permitAll()
                .requestMatchers("/api/v1/admin/**").hasRole("ADMIN")
                .anyRequest().authenticated())
            .addFilterBefore(tokenAuthFilter, UsernamePasswordAuthenticationFilter.class);
        return http.build();
    }

    @Bean
    public CorsConfigurationSource corsConfigurationSource() {
        CorsConfiguration config = new CorsConfiguration();
        config.setAllowedOrigins(List.of("http://localhost:4200"));
        config.setAllowedMethods(List.of("GET", "POST", "PUT", "DELETE", "OPTIONS"));
        config.setAllowedHeaders(List.of("*"));
        config.setAllowCredentials(true);
        UrlBasedCorsConfigurationSource source = new UrlBasedCorsConfigurationSource();
        source.registerCorsConfiguration("/**", config);
        return source;
    }
}
```

> **注意：** `hasRole("ADMIN")` 会在数据库中 `role` 字段值前自动追加 `ROLE_` 前缀对比。
> 数据库存的是 `admin`，所以实现 UserDetails 时需要返回 `ROLE_ADMIN`。

## 六、数据库表

### 6.1 新增 `user_tokens` 表

```sql
CREATE TABLE user_tokens (
    id          BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id     BIGINT       NOT NULL COMMENT '用户ID',
    token_hash  VARCHAR(64)  NOT NULL COMMENT 'SHA-256(token)',
    token_type  VARCHAR(10)  NOT NULL COMMENT 'ACCESS / REFRESH',
    expires_at  DATETIME     NOT NULL COMMENT '过期时间',
    created_at  DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    revoked     TINYINT(1)   NOT NULL DEFAULT 0 COMMENT '是否吊销',
    replaced_by VARCHAR(64)  DEFAULT NULL COMMENT 'rotation: 被哪个新 token_hash 替换',
    CONSTRAINT fk_token_user FOREIGN KEY (user_id) REFERENCES users(id)
);

CREATE UNIQUE INDEX uk_token_hash ON user_tokens(token_hash);
CREATE INDEX idx_user_id ON user_tokens(user_id);
CREATE INDEX idx_expires ON user_tokens(expires_at);
```

### 6.2 Domain 实体

```java
@Data
public class UserToken {
    private Long id;
    private Long userId;
    private String tokenHash;
    private String tokenType;
    private LocalDateTime expiresAt;
    private LocalDateTime createdAt;
    private Boolean revoked;
    private String replacedBy;
}
```

### 6.3 Domain 变更: `Users.java` — **无需改动**

现有 `Users` 实体字段齐全（username, passwordHash, role, status），无需修改。

> **密码升级策略：** 当前数据库中的 `password_hash` 可能是明文/MD5。为防止切换后现有用户无法登录，
> 在 `UserDetailsServiceImpl` 中校验密码时，使用 `DelegatingPasswordEncoder` 兜底：
> 1. 默认用 `bcrypt` 匹配
> 2. 匹配失败时尝试 `NoOpPasswordEncoder`（明文兼容）
> 3. 明文匹配成功后 → 自动用 bcrypt 重编码并更新数据库
> 这样无需迁移脚本，旧密码在下次登录时自动升级。

## 七、核心流程时序

### 7.1 登录

```
Client                     Server
  │                          │
  │  POST /auth/login        │
  │  {username, password}    │
  │ ──────────────────────►  │
  │                          │ UsernamePasswordAuthenticationFilter
  │                          │ → UserDetailsServiceImpl.loadUserByUsername()
  │                          │ → BCryptPasswordEncoder.matches()
  │                          │ → LoginSuccessHandler
  │                          │   → TokenGenerator.generate() × 2
  │                          │   → TokenService.issueTokenPair()
  │                          │   → DbTokenRepository.save() × 2
  │  TokenPair {             │
  │    access_token,         │
  │    refresh_token,        │
  │    expires_in            │
  │  }                       │
  │ ◄──────────────────────  │
```

### 7.2 访问受保护资源

```
Client                     Server
  │                          │
  │  GET /api/v1/auth/me     │
  │  Authorization: Bearer xY7vE3k...  │
  │ ──────────────────────►  │
  │                          │ TokenAuthenticationFilter
  │                          │ → 提取 header 中的 token
  │                          │ → SHA-256 计算 hash
  │                          │ → DbTokenRepository.findByHash()
  │                          │ → 检查 revoked & expires_at
  │                          │ → 构建 UsernamePasswordAuthenticationToken
  │                          │ → SecurityContextHolder.setAuthentication()
  │                          │
  │                          │ Controller 处理
  │                          │ → SecurityContextHolder 取用户
  │                          │
  │  UserInfoVO              │
  │ ◄──────────────────────  │
```

### 7.3 Refresh 续期

```
Client                     Server
  │                          │
  │  POST /auth/refresh      │
  │  {refresh_token}         │
  │ ──────────────────────►  │
  │                          │ TokenService.refreshAccessToken()
  │                          │ → SHA-256 计算 hash
  │                          │ → 查 DbTokenRepository.findByHash()
  │                          │ → 检查: 未过期、未吊销、type=REFRESH
  │                          │ → Rotation: 旧 refresh_token → revoked=1, replaced_by=新hash
  │                          │ → 签发新的 TokenPair (新 access + 新 refresh)
  │                          │ → DbTokenRepository.save() × 2
  │                          │
  │  TokenPair (新)          │
  │ ◄──────────────────────  │
```

> **多 tab 并发处理：** 如果 refresh_token 已 `revoked=1` 且有 `replaced_by`，
> 说明是被另一个请求 rotation 掉的。此时不直接返回失败，而是尝试用
> `replaced_by` 再次查找——如果该 token 仍有效则签发新的 token pair。
> 这样多 tab 下只有一个 refresh 真正成功，但其他 tab 也不会丢失登录态。

## 十、前端联动方案

### 10.1 存储变更

当前前端在 `sessionStorage` 中存单个 token（key: `lblog_token`），需要改为：

```
sessionStorage.setItem('lblog_access_token', '<access_token>');
sessionStorage.setItem('lblog_refresh_token', '<refresh_token>');
```

### 10.2 axios 拦截器

新增 401 自动续期逻辑：

```
axios 响应拦截器:
  └→ 收到 401
      └→ 判断是否已在刷新中（用 isRefreshing 锁）
          ├→ 否 → 调 POST /auth/refresh
          │   └→ 成功 → 更新本地 token → 重放原请求
          │   └→ 失败 → 清除本地 token → 跳转登录
          └→ 是 → 将原请求加入等待队列
                    → 刷新完成后依次重放
```

> 刷新接口本身不应触发 401 拦截器（循环调用），需要在拦截器中排除 `/auth/refresh`。

### 10.3 API 工具函数变更

```typescript
// 当前: 单 token
function authHeader(): Record<string, string> {
  const token = sessionStorage.getItem('lblog_token');
  return token ? { Authorization: `Bearer ${token}` } : {};
}

// 目标: 从 access_token 取
function authHeader(): Record<string, string> {
  const token = sessionStorage.getItem('lblog_access_token');
  return token ? { Authorization: `Bearer ${token}` } : {};
}
```

### AdminController 改动

- 当前: `private static final Long TEST_USER_ID = 1L`
- 目标: 从 `((LoginUser) SecurityContextHolder...).getUserId()` 获取真实用户 ID

## 十二、扩展到 Redis 的迁移路径

未来迁移到 Redis 时，变更范围极窄：

### 新增文件

```
security/repository/RedisTokenRepository.java
config/TokenStorageConfig.java          — @ConditionalOnProperty 切换
```

### TokenRecord 增加 Redis 兼容字段

```java
// 新增字段，MySQL 实现忽略，Redis 实现使用
private Long ttlSeconds;  // Redis TTL 秒数
```

### 切换方式

```yaml
# application.yml
lblog:
  token:
    storage: redis   # db | redis
```

```java
@Configuration
public class TokenStorageConfig {
    @Bean
    @ConditionalOnProperty(value = "lblog.token.storage", havingValue = "redis", matchIfMissing = false)
    public TokenRepository redisTokenRepository() {
        return new RedisTokenRepository();
    }

    @Bean
    @ConditionalOnProperty(value = "lblog.token.storage", havingValue = "db", matchIfMissing = true)
    public TokenRepository dbTokenRepository() {
        return new DbTokenRepository();
    }
}
```

### Redis 存储结构

```
Key: token:{sha256_hash}
Value: {"userId": 1, "type": "ACCESS", "userId": 1, "createdAt": ...}

Key: user_tokens:{userId}
Type: Set
Value: {sha256_hash1, sha256_hash2, ...}  — 用于查询用户所有 token

过期: 直接用 Redis TTL，无需定时任务
```
