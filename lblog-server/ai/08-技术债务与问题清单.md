## 🟡 高优先级改进

**Tool 与传输层耦合。** `DisplayDrawTool` 通过 `ToolContext` 拿 `SseEmitter`，工具方法和 SSE 捆在一起。当前只有一个 Agent、一个 tool，耦合未造成实际 bug。等加第二个 Agent 时再解耦。

---

**DrawService 做了太多事。** 传输层（心跳 + SSE）已剥离为 `AgentStreamTransport`。剩余编排骨架（调 ChatClient、处理 chunk）仍在 DrawService。等加第二个 Agent 时提取 `AgentExecutor` 基类。

---

**Token 估算精度低。** `CharBasedTokenEstimator` 用 `text.length() / 2 + 1`。中文和英文的实际 token 数差异大，预估可能偏高或偏低 50%。`CompressionAdvisor` 基于这个估算做压缩决策。

建议：引入 jtokkit，或至少在估算结果上乘 1.2 安全系数。

---

**DrawPromptManager 模型路由还在代码里。** 提示词内容已经外置到 DB，但"什么模型用哪套提示词"还在 Java 里用 `modelId.contains("claude-opus-4-5")` 判断。

建议：在 `site_config` 表建 `model.prompt.override.{modelName}` key，PromptManager 查配置决定用哪套提示词。

---

**Skill Admin API 未实现。** 增删改 skill 包只能直接操作数据库。`AdminPromptController` 9 个接口齐全，Skill 一个没有。

建议：参照 AdminPromptController 范式，新增 AdminSkillController：列表、新增、更新、停用、reload 缓存。

---

## 🟢 低优先级改进

**4 个未使用的 Service 方法。** `saveUserMessage()`、`saveAssistantMessage()`、`saveToolMessage()`、`loadRecentMessages()` 定义了但无人调用。ChatHistoryAdvisor 全部走 `batchSave()`。

**会话标题硬编码。** `"New draw session"`，不含任何上下文信息。会话列表里全是同名字，区分不了。

**listSessions 无总数。** 返回 `List<V>` 而非 `PageResult<V>`，前端不知道总共多少页。

**tool_calls JSON 未反序列化到 VO。** `convertToVOList()` 没设 toolCalls 字段，前端看不到工具调用历史。

**缺少异常重试和熔断。** DeepSeek API 不可用时直接 `emitter.completeWithError()`，无指数退避、无熔断。

**Seed 幂等但不会更新。** 文件改了再 seed 不生效。建议 seed 时比较 checksum，文件变更时自动更新。

**有效时间字段未实现。** `ai_prompts.effective_from / effective_to` 建了但 `AiPromptService` 忽略。

**reload 缓存全量清空。** 多模块时 draw 的变更导致 chat 缓存也被清。

**FilePromptLoader 无路径校验。** module/key 参数无字符白名单。

**SlidingWindowStrategy.load() 走全量 DB 查询。** 取最后 20 条却先查全量。

**LoadSkillTool 无加载统计。** 不记录谁加载了哪个 skill。

**Skill 无初始种子数据。** DB 里没有 skill 记录时 LLM 什么都加载不到。

**DrawRateLimiter 限制值硬编码。** `MAX_REQUESTS_PER_MINUTE = 10` 写死在代码里。

**SseEmitter 低版本线程安全。** 低版本 Spring 的 SseEmitter.send() 并发时可能状态错乱。当前单线程 send 不受影响。

**Spring AI 适配文件。** 4 个文件等官方修复后可删除。每升级一版检查一次。

**CompressionAdvisor 无指标。** 不知道压缩频率、节省 token 量。

## 优先修复建议

按"用户可见 + 改动量小"排序：

1. previewText 为空 — 前端可见 Bug，改一行 SQL
2. getOrCreateSession 总是创建 — 用户可见 Bug，加一行查询
3. Skill 初始种子数据 — 没数据 LLM 无法加载技能
4. Skill Admin API — 不能每次改 skill 都直连 DB
5. 会话标题硬编码 — 用户体验问题
6. Token 估算精度 — 影响压缩准确度
