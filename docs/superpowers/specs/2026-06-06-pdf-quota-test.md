## 场景 1: 新用户首次上传（无配额记录）

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 1.1 | 确保 `pdf_user_quota` 中无该用户记录 | `SELECT * FROM pdf_user_quota WHERE user_id = ?` 返回空 |
| 1.2 | 使用该用户上传 PDF | **返回 400/403，提示"请先申请 PDF 上传权限"** |
| 1.3 | 查表确认自动插入默认记录 | `SELECT * FROM pdf_user_quota WHERE user_id = ?` 返回 `quota_bytes=524288000, allow_upload=0` |
| 1.4 | 再次上传 | 提示 "PDF 上传权限未开启，请联系管理员" |

## 场景 2: 管理员开启上传权限

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 2.1 | Admin 访问 `GET /api/v1/admin/pdf-quotas` | 列表包含该用户，allowUpload=0 |
| 2.2 | `PUT /api/v1/admin/pdf-quotas/{userId}/allow-upload?allowUpload=1` | 返回 200 |
| 2.3 | 用户再次上传 < 500MB 的 PDF | **上传成功** |
| 2.4 | 侧边栏查看用量 | 显示已用空间 + 进度条 |

## 场景 3: 配额超限

> 注：最低配额限制为 1 MB（1048576 字节）

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 3.1 | Admin 设置配额为 1MB: `PUT /.../quota?quotaBytes=1048576` | 返回 200 |
| 3.2 | 上传一个 2MB 的 PDF（当前已用+2MB > 1MB） | **返回 400，提示"已用 X MB / 配额 1.0 MB"** |
| 3.3 | 上传一个 0.5MB 的 PDF（不超配额） | 上传成功 | |

## 场景 4: 管理员禁用上传

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 4.1 | `PUT /.../allow-upload?allowUpload=0` | 返回 200 |
| 4.2 | 用户上传 PDF | **提示 "PDF 上传权限未开启"** |

## 场景 5: 管理员修改配额

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 5.1 | `PUT /.../quota?quotaBytes=104857600` (100MB) | 返回 200 |
| 5.2 | 用户 `GET /api/v1/pdf/stats` | quotaBytes=104857600 |
| 5.3 | 前端管理页刷新 | 配额列显示 100 MB |

## 场景 6: 非法参数校验

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 6.1 | `PUT /.../quota?quotaBytes=0` | 400 "配额不能小于 1 MB" |
| 6.2 | `PUT /.../quota?quotaBytes=-100` | 400 "配额不能小于 1 MB" |
| 6.3 | `PUT /.../allow-upload?allowUpload=2` | 400 "allowUpload 必须为 0 或 1" |
| 6.4 | `PUT /.../allow-upload?allowUpload=-1` | 400 |

## 场景 7: 权限校验

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 7.1 | 非 admin 用户请求 `GET /api/v1/admin/pdf-quotas` | **403 Forbidden** |
| 7.2 | 非 admin 用户 `PUT /.../quota` | **403** |
| 7.3 | 用户 A 删除用户 B 的 PDF: `DELETE /api/v1/pdf/files/{B的id}` | **400/403 文件不存在**（所有权校验） |

## 场景 8: 前端管理页面

| 步骤 | 操作 | 预期结果 |
