# 内容创作接口文档

## 一、鉴权标准

### 1.1 认证方式

采用 **Bearer Token**（JWT）进行接口鉴权。

| 项目 | 说明 |
|------|------|
| Token 类型 | JWT |
| 传递方式 | `Authorization: Bearer <token>` 请求头 |
| Token 有效期 | 7 天 |
| 存储方式 | 前端存 `sessionStorage`，每次请求携带 |

### 1.2 未登录响应

所有需要鉴权的接口，若未携带有效 Token，统一返回：

```json
{
  "code": 401,
  "message": "未登录或 Token 已过期",
  "data": null
}
```

### 1.3 Token 验证失败

```json
{
  "code": 401,
  "message": "Token 无效",
  "data": null
}
```

### 1.4 无权限（角色不足）

```json
{
  "code": 403,
  "message": "无权限访问",
  "data": null
}
```

## 三、登录相关接口

### 3.1 用户登录

登录成功后返回 JWT Token，前端存储后在后续请求中携带。

- **接口：** `POST /api/v1/auth/login`
- **鉴权：** 无需 Token

**请求体：**

```json
{
  "username": "admin",
  "password": "admin123"
}
```

**响应（成功）：**

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiJ9...",
    "user": {
      "id": 1,
      "username": "admin",
      "nickname": "站长",
      "avatar": "https://...",
      "email": "admin@lblog.com",
      "role": "admin"
    }
  }
}
```

**响应（失败）：**

```json
{
  "code": 401,
  "message": "用户名或密码错误",
  "data": null
}
```

### 3.2 获取当前用户信息

用于前端验证 Token 是否有效、获取当前登录用户信息。

- **接口：** `GET /api/v1/auth/me`
- **鉴权：** 需要 Bearer Token

**请求头：**

```
Authorization: Bearer eyJhbGciOiJIUzI1NiJ9...
```

**响应（成功）：**

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "id": 1,
    "username": "admin",
    "nickname": "站长",
    "avatar": "https://...",
    "email": "admin@lblog.com",
    "role": "admin"
  }
}
```

**响应（Token 无效/过期）：**

```json
{
  "code": 401,
  "message": "Token 已过期，请重新登录",
  "data": null
}
```

### 3.3 退出登录

- **接口：** `POST /api/v1/auth/logout`
- **鉴权：** 需要 Bearer Token

**请求体：** 无

**响应：**

```json
{
  "code": 0,
  "message": "success",
  "data": null
}
```

> 服务端可将 Token 加入黑名单，前端直接清除本地 Token 即可。

## 四、前端 Header 工具

```typescript
// api.ts — 请求时自动携带 Token
function authHeader(): Record<string, string> {
  const token = sessionStorage.getItem('lblog_token');
  return token ? { Authorization: `Bearer ${token}` } : {};
}
```
