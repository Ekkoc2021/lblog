# 写文章相关接口文档

## 一、说明

### 1.1 鉴权方式

所有 `/api/v1/admin/**` 接口均需 Bearer Token 鉴权，鉴权标准与[内容创作接口](./4内容创作接口.md)一致。

- Token 存放：`sessionStorage`，key 为 `lblog_token`
- 请求头：`Authorization: Bearer <token>`
- 未登录统一返回：`{ "code": 401, "message": "未登录或 Token 已过期", "data": null }`

### 1.2 通用响应格式

```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

### 1.3 API 前缀

```
基础路径: http://localhost:8099/iblogserver/api/v1
```

## 二、文章管理

### 2.1 获取管理端文章列表

管理后台文章表格展示，需包含草稿和私密文章。

- **接口：** `GET /api/v1/admin/posts`
- **鉴权：** 需要 Bearer Token

**请求参数：**

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| page | int | 否 | 1 | 页码 |
| pageSize | int | 否 | 10 | 每页条数 |
| status | int | 否 | — | 筛选：0-草稿，1-已发布，2-私密，不传查全部 |
| keyword | string | 否 | — | 搜索标题（模糊匹配） |

**排序：** 默认按 `updated_at` 降序排列。

**响应示例：**

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "list": [
      {

          { "id": 1, "name": "Java", "slug": "java" }

        "viewCount": 1256,
        "likeCount": 89,
        "commentCount": 23,
        "commentEnable": 1,
```

### 2.3 创建文章

- **接口：** `POST /api/v1/admin/posts`
- **鉴权：** 需要 Bearer Token

**请求体：**

```json
{
  "title": "文章标题",
  "slug": "article-slug",
  "excerpt": "文章摘要",
  "body": "## Markdown 正文内容...",
  "featuredImage": null,
  "status": 0,
  "categoryId": 1,
  "tagIds": [1, 3],
  "seriesId": null,
  "commentEnable": 1
}
```

**请求体字段说明：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| title | string | 是 | 文章标题 |
| slug | string | 是 | URL 别名，需唯一 |
| excerpt | string | 否 | 摘要，不传则由后端从 body 截取前 200 字 |
| body | string | 是 | Markdown 正文 |
| featuredImage | string | 否 | 特色图片 URL |
| status | int | 是 | 0-草稿，1-已发布，2-私密 |
| categoryId | long | 否 | 所属分类 ID |
| tagIds | long[] | 否 | 标签 ID 列表 |
| seriesId | long | 否 | 关联专栏 ID |
| commentEnable | int | 否 | 0-禁止评论，1-允许评论（默认 1） |

**响应示例：**

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "id": 10
  }
}
```

**补充逻辑：**

- 创建时 `status=1`（已发布），自动设置 `published_at = now()`
- `slug` 重复返回 `{ "code": 400, "message": "URL 别名已存在" }`
- `body` 写入 `post_contents` 表
- 如果传了 `tagIds`，写入 `post_tags` 关联表
- 如果传了 `seriesId`，写入 `series_posts` 关联表

---

### 2.4 更新文章

- **接口：** `PUT /api/v1/admin/posts/{id}`
- **鉴权：** 需要 Bearer Token

**请求体（与创建相同，支持部分字段更新）：**

```json
{
  "title": "新标题",
  "slug": "new-slug",
  "excerpt": "新摘要",
  "body": "新的 Markdown 正文...",
  "featuredImage": null,
  "status": 1,
  "categoryId": 2,
  "tagIds": [2, 4],
  "seriesId": 1,
  "commentEnable": 1
}
```

**补充逻辑：**

- 所有字段均为可选，只传需要更新的字段
- 如果 `status` 从非 1 变为 1，设置 `published_at = now()`
- 如果传了 `tagIds`，先删后插 `post_tags`（全量替换）
- 如果传了 `seriesId`，同步 `series_posts` 关联
- 如果传了 `body`，更新 `post_contents` 表

---

### 2.5 删除文章（软删除）

- **接口：** `DELETE /api/v1/admin/posts/{id}`
- **鉴权：** 需要 Bearer Token

**响应：**

```json
{
  "code": 0,
  "message": "success",
  "data": null
}
```

**补充逻辑：**

- 逻辑删除：设置 `deleted_at = now()`
- 同时级联删除关联的 `post_tags`、`series_posts` 记录（逻辑删除）

---

### 2.6 校验 Slug 是否可用

编辑器保存前实时校验 slug 是否被占用。

- **接口：** `GET /api/v1/admin/posts/check-slug`
- **鉴权：** 需要 Bearer Token

**请求参数：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| slug | string | 是 | 要检查的 URL 别名 |
| excludeId | long | 否 | 编辑时排除自身 ID |

**响应：**

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "available": true
  }
}
```

## 六、图片上传

### 6.1 上传图片

Markdown 编辑器中粘贴或点击上传图片。

- **接口：** `POST /api/v1/upload/image`
- **鉴权：** 需要 Bearer Token
- **请求格式：** `multipart/form-data`

**请求参数：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| file | File | 是 | 图片文件 |

**响应：**

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "url": "/uploads/2026/04/25/abc123.png"
  }
}
```

**补充说明：**

- 支持格式：jpg、png、gif、webp、svg
- 单文件大小限制：5MB
- 存入本地 `uploads/` 目录，按日期分目录存储
- 返回的 URL 为相对路径，前端拼装完整地址：`http://localhost:8099/iblogserver/uploads/...`

## 八、前端对接清单

### 8.1 管理页面初始化数据加载

编辑器和管理页面需要加载以下数据填充下拉选择框：

| 接口 | 方法 | 用途 | 使用页面 |
|------|------|------|----------|
| `/api/v1/categories` | GET | 分类下拉选项 | PostEditor / SeriesManage |
| `/api/v1/tags` | GET | 标签下拉选项 | PostEditor |
| `/api/v1/series` | GET | 专栏下拉选项 | PostEditor |

> 这三个是已有接口，无需新增。管理页面需获取**全部**数据（不传 `limit` 或传入足够大的值），避免下拉选项缺失。

### 8.2 保存/发布流程

#### 新建文章

```
1. 用户填写标题 + Markdown 正文
2. 点击「保存草稿」或「发布」→ 弹出元信息弹窗
3. 弹窗中：分类/标签/专栏下拉框数据来自 8.1（已加载完毕）
4. 用户填写/修改元信息 → 点击确定
5. POST /api/v1/admin/posts → 创建文章（含 body、tagIds、seriesId）
6. 创建成功 → 清除本地草稿 → 跳转 /admin/posts
```

#### 编辑已有文章

```
1. 进入编辑器时：
   ├── GET /api/v1/admin/posts/{id} → 加载文章数据
   └── GET /api/v1/categories + tags + series → 加载下拉选项
2. 修改标题/正文/元信息
3. 点击保存 → PUT /api/v1/admin/posts/{id}
4. 更新成功 → 跳转 /admin/posts
```
