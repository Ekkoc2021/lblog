# LBlog 项目开发计划

## 一、项目概述

LBlog 是一个**单用户个人技术博客**，采用前后端分离架构。当前已完成项目骨架搭建和首页列表展示，后续按以下计划迭代开发。

---

## 二、技术栈

| 层级 | 技术 | 版本 |
|------|------|------|
| 前端框架 | React | 19 |
| 开发语言 | TypeScript | 6 |
| 构建工具 | Vite | 8 |
| UI 组件库 | Ant Design | 6 |

| ORM | MyBatis | 3.x |
| 数据库 | MySQL | 8.x |
| 分页 | PageHelper | 2.x |
| 接口文档 | SpringDoc OpenAPI | 2.x |

---

## 三、架构决策记录

以下决策经确认后写入计划，后续开发遵循：

| 决策点 | 结论 | 原因 |
|--------|------|------|
| 文章 URL | `/posts/:slug` | 语义化 URL，对 SEO 友好 |
| Markdown 渲染 | 客户端 react-markdown | 前端解析，后端只需返回原始 Markdown 字符串 |
| 图片上传 | 本地目录存储 | 单用户博客，简单够用 |
| 评论嵌套 | 两级嵌套（仅回复根评论） | 体验良好且复杂度可控 |
| 用户定位 | 单用户个人博客 | 登录注册暂缓，优先打磨阅读体验 |
| 分页风格 | 加载更多 | 保持操作连贯性 |
| 组件抽取时机 | 做详情页之前先抽 | 避免后续重复开发 |
| 专栏文章接口 | 后端新增独立接口 | 与通用文章列表接口职责分离 |

**状态：** ⬜ 待处理

### 第三阶段：用户互动（P2）

#### 3.1 点赞功能

**交互：**
- 文章详情页点赞按钮，点击切换状态
- 未点赞：空心图标 + 灰色
- 已点赞：实心图标 + 蓝色 + 微动效（+1 动画）
- 点赞数实时更新

**数据存储：**
- 单用户模式，点赞记录存在 `like_records` 表
- 未登录时用 IP + User-Agent 做指纹标识（简单防刷）

**后端需新增：**
- `POST /api/v1/posts/{id}/like` — 点赞
- `DELETE /api/v1/posts/{id}/like` — 取消点赞
- `like_records` 表

---

#### 3.2 评论系统（两级嵌套）

**数据模型：**
```
comments
├── id (PK)
├── post_id (关联文章)
├── parent_id (父评论ID，NULL=根评论，非NULL=回复)
├── author (评论者昵称)
├── email (评论者邮箱，用于 Gravatar 头像)
├── body (评论内容)
├── created_at
└── deleted_at (软删除)
```

**交互说明：**
- 根评论下方显示"回复"按钮
- 点击"回复"在根评论下方展开回复输入框
- 回复的内容缩进显示在根评论下方
- 根评论按时间倒序，回复按时间正序

**涉及文件：**
- 新建 `src/components/CommentSection.tsx`

**后端需新增：**
- `comments` 表及对应 Mapper/Service/Controller
- `GET /api/v1/posts/{id}/comments` — 评论列表（分页）
- `POST /api/v1/posts/{id}/comments` — 发表评论
- 评论需审核后才展示（可配置，默认自动通过）

### 第四阶段：内容创作（P3）

> 单用户模式，不涉及登录权限校验，编辑器和管理页面通过隐藏路由或简单密码访问。

**技术选型：**
- `bytemd`（基于 ByteMD，Ant Design 风格，对 React 友好）

**编辑器功能：**
- 全功能工具栏（标题、加粗、斜体、列表、代码块、引用、表格、图片、链接）
- 分栏编辑：左侧编辑，右侧实时预览
- 全屏模式
- 快捷键（Ctrl+S 保存）

#### 4.2 文章发布/管理

**路由：**
- `/editor` — 新建文章
- `/editor/:id` — 编辑已有文章

**功能：**
- 文章标题输入
- 分类下拉选择
- 标签多选（支持输入新标签）
- Markdown 正文编辑
- 封面图上传
- 开关：允许评论、是否私密
- 操作：保存草稿 / 立即发布

**管理页面：**
- `/admin/posts` — 文章列表管理（查看/编辑/删除/切换状态）

**后端需新增：**
- `POST /api/v1/posts` — 创建文章
- `PUT /api/v1/posts/{id}` — 更新文章
- `DELETE /api/v1/posts/{id}` — 删除文章（软删除）
- `GET /api/v1/admin/posts` — 管理端文章列表（含草稿/私密）
- `POST /api/v1/upload/image` — 图片上传
