# LBlog 项目开发计划

## 一、项目概述

LBlog 是一个**单用户个人技术博客**，采用前后端分离架构。当前已完成项目骨架搭建和首页列表展示，后续按以下计划迭代开发。

---

## 三、架构决策记录

以下决策经确认后写入计划，后续开发遵循：

| 决策点 | 结论 | 原因 |
|--------|------|------|
| 文章 URL | `/posts/:slug` | 语义化 URL，对 SEO 友好 |
| Markdown 渲染 | 客户端 react-markdown | 前端解析，后端只需返回原始 Markdown 字符串 |
| 图片上传 | 本地目录存储 | 单用户博客，简单够用 |
| 评论嵌套 | 两级嵌套（仅回复根评论） | 体验良好且复杂度可控 |
| 用户定位 | 单用户个人博客 | 登录注册暂缓，优先打磨阅读体验 |
| 分页风格 | 加载更多 | 保持操作连贯性 |
| 组件抽取时机 | 做详情页之前先抽 | 避免后续重复开发 |
| 专栏文章接口 | 后端新增独立接口 | 与通用文章列表接口职责分离 |

**图片粘贴自动上传（方案已定，待实现）：**

```
流程：
1. 监听 TextArea paste 事件
2. 检查 clipboardData.files 中是否有 image 类型
3. 有图片 → 阻止默认粘贴 → 调用上传接口
4. 上传完成后在光标位置插入 ![](url)
5. 支持多张连续粘贴（批量上传 + 进度提示）

关键设计点：
- 光标位置：通过 textarea.selectionStart/End 确定插入点
- 上传反馈：底部提示"正在上传图片（1/3）..."
- 与「插入图片」按钮复用同一上传函数
- 需前端图片大小校验（与后端约定上限）
```

**后端接口：**
- `POST /api/v1/upload/image` — `multipart/form-data`，返回图片 URL
