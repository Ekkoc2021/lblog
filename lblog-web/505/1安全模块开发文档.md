# 安全模块开发文档

> 基于 Token（Access + Refresh）的双令牌认证体系，数据库存储 SHA-256 哈希，无状态会话。

---

## 一、整体架构

```
┌──────────────┐     ┌───────────────────┐     ┌───────────────┐
│              │     │  SecurityFilterChain │     │               │
│  客户端      │────▶│  ┌─────────────────┐│────▶│  Controller   │
│  (Browser)   │     │  │ TokenAuthFilter ││     │  (业务逻辑)    │
│              │     │  └─────────────────┘│     │               │
└──────────────┘     │  ┌─────────────────┐│     └───────────────┘
  Bearer Token       │  │ ExceptionHandler││
                     │  └─────────────────┘│
                     │  ┌─────────────────┐│
                     │  │ AuthEntryPoint  ││  → 401 JSON
                     │  │ AccessDenied    ││  → 403 JSON
                     │  └─────────────────┘│
                     └───────────────────┘
```

**请求流程**:
1. 客户端发送 `Authorization: Bearer <access_token>`
2. `TokenAuthenticationFilter` 提取 token → SHA-256 查库 → 校验未吊销/未过期/类型=ACCESS → 设 SecurityContext
3. `SecurityFilterChain` 按 URL 匹配鉴权规则
4. 通过 → Controller；失败 → CustomAuthEntryPoint(401) 或 CustomAccessDeniedHandler(403)

## 三、核心组件说明

### 3.1 SecurityConfig

```java
// 关键配置点
.authorizeHttpRequests(auth -> auth
    // 公开接口（无需登录）—— 按实际 Controller 路径填写
    .requestMatchers(POST, "/api/v1/auth/login").permitAll()
    .requestMatchers(POST, "/api/v1/auth/refresh").permitAll()
    .requestMatchers(GET, "/api/v1/posts/**").permitAll()
    .requestMatchers(GET, "/api/v1/categories/**").permitAll()
    // ... 其他公开接口
    // 静态资源
    .requestMatchers(GET, "/uploads/**").permitAll()
    // 管理接口 —— hasRole("ADMIN") = 需要 ROLE_ADMIN 权限
    .requestMatchers("/api/v1/admin/**").hasRole("ADMIN")
    // 其余全部需要认证
    .anyRequest().authenticated()
)
```

**⚠️ 常见坑**: `permitAll()` 的路径必须与 Controller 的 `@RequestMapping` 一致。Controller 用 `@RequestMapping("/api/v1")` 则写 `/api/v1/**`，不是 `/api/v1/home/**`。

### 3.2 TokenAuthenticationFilter

- 实现 `OncePerRequestFilter`，每次请求执行一次
- 从 Header 提取 `Bearer xxx`
- 调用 `TokenService.validateAccessToken()` 验证
- 验证通过 → 设 `UsernamePasswordAuthenticationToken` 到 `SecurityContextHolder`
- 验证不通过 → 清除 Context，**继续** filter chain（由后面的 ExceptionTranslationFilter 处理）

### 3.3 CustomAuthEntryPoint + CustomAccessDeniedHandler

- **401**: `{"code":401,"message":"未登录或 Token 已过期"}`
- **403**: `{"code":403,"message":"无权限访问"}`

均为 JSON 格式，非重定向。

### 3.4 LoginUser (UserDetails)

```java
// 从 Users 实体构造
new LoginUser(usersEntity);

// getAuthorities() 返回
"ROLE_" + role.toUpperCase()  // 如 "ROLE_ADMIN", "ROLE_AUTHOR"
```

`isEnabled()` = `status == 1`。这决定了禁用用户能否登录。

### 3.5 TokenGenerator

```java
// 生成令牌对
TokenPairRaw pair = tokenGenerator.generate();
// pair.rawAccessToken  — 返回给客户端的明文
// pair.rawRefreshToken — 返回给客户端的明文  
// pair.accessHash       — SHA-256 存库
// pair.refreshHash      — SHA-256 存库

// 对已有 token 算哈希
String hash = tokenGenerator.hash(rawToken);
```

### 3.6 TokenRepository (接口) + DbTokenRepository (实现)

```java
void save(TokenRecord);              // 存储新 token
Optional<TokenRecord> findByHash();  // 按 hash 查找
Optional<TokenRecord> findByHashForUpdate();  // FOR UPDATE 行锁（refresh 并发控制）
void revoke(hash);                   // 吊销
void revokeAllByUserId(userId);      // 吊销用户所有 token
void updateReplacedBy(hash, replacedBy);  // Refresh Rotation 链记录
```

### 3.7 UserDetailsServiceImpl

```java
// 登录时被 AuthenticationManager 调用
loadUserByUsername(username) → 查 users 表 → 返回 LoginUser
// 注意：密码比较由 DaoAuthenticationProvider 完成
```

## 四、Token 生命周期

### 4.1 签发（登录成功）

```
AuthController.login()
  → authenticationManager.authenticate()   // 验证用户名密码
    → UserDetailsServiceImpl.loadUserByUsername()
    → PasswordEncoder.matches() 检查密码
  → tokenService.issueTokenPair(userId)
    → tokenGenerator.generate()            // 生成随机 token + SHA-256
    → tokenRepository.save(access_record)  // 存库
    → tokenRepository.save(refresh_record)
    → 返回 TokenPairVO(accessToken, refreshToken, expiresIn, user)
```

### 4.2 验证（每次请求）

```
TokenAuthenticationFilter.doFilter()
  → extractToken(request)  // 取 Authorization: Bearer xxx
  → tokenService.validateAccessToken(rawToken)
    → hash(rawToken)
    → findByHash(hash)
    → 校验: 未 revoked, 未 expired, type=ACCESS
    → 查 users 表获取用户信息
    → 返回 LoginUser
  → 设 SecurityContext
```

### 4.3 刷新

```
AuthController.refresh()
  → tokenService.refreshAccessToken(rawRefreshToken)
    → hash + findByHashForUpdate(FOR UPDATE 行锁)
    → 有效: 签发新对, 原 token revoked + set replaced_by
    → 已 revoked + 有 replaced_by: 链式查找(多 tab 兜底)
    → 返回新 TokenPairVO
```

**Refresh Rotation 链式机制**:
```
原始 refresh ──(refresh)──▶ 新 refresh ──(refresh)──▶ 最新 refresh
     │                            │
     └── 再次请求 ──▶ replaced_by 查找 ──▶ 可用
```

### 4.4 吊销

```java
// 单个 token
tokenService.revokeToken(rawToken);
  → hash → tokenRepository.revoke(hash)

// 用户全部 token（改密时）
tokenService.revokeAllUserTokens(userId);
  → tokenRepository.revokeAllByUserId(userId)
```

### 4.5 定时清理

`TokenCleanupTask` 每天凌晨 3 点执行，删除 `已过期 + 已吊销` 的 token 记录。

## 五、如何开发带权限的接口

### 5.1 公开接口（无需登录）

```java
@GetMapping("/posts")
public ApiResponse<?> list() {
    return ApiResponse.success(service.list());
}
```
- 无需在 Controller 做任何认证操作
- 在 `SecurityConfig` 的 `permitAll()` 中添加路径

### 5.2 需登录接口

```java
@PostMapping("/auth/logout")
public ApiResponse<Void> logout() {
    Authentication auth = SecurityContextHolder.getContext().getAuthentication();
    if (auth != null && auth.getCredentials() instanceof String) {
        String rawToken = (String) auth.getCredentials();
        tokenService.revokeToken(rawToken);
    }
    return ApiResponse.success(null);
}
```

获取当前用户:
```java
Authentication auth = SecurityContextHolder.getContext().getAuthentication();
if (auth != null && auth.getPrincipal() instanceof LoginUser) {
    LoginUser user = (LoginUser) auth.getPrincipal();
    Long userId = user.getUserId();
    String role = user.getRole();
}
```

### 5.3 需 Admin 角色接口

```java
// 方式一：SecurityConfig 统一配置（推荐）
.requestMatchers("/api/v1/admin/**").hasRole("ADMIN")

// 方式二：方法级注解
@PreAuthorize("hasRole('ADMIN')")
@GetMapping("/admin/something")
```

`hasRole("ADMIN")` 实际校验的是 `ROLE_ADMIN`（Spring Security 自动加前缀）。

### 5.4 获取当前用户工具方法

```java
private Long getCurrentUserId() {
    Authentication auth = SecurityContextHolder.getContext().getAuthentication();
    if (auth != null && auth.getPrincipal() instanceof LoginUser) {
        return ((LoginUser) auth.getPrincipal()).getUserId();
    }
    return null;
}
```

### 5.5 公开接口中的访客标识

```java
@PostMapping("/posts/{id}/like")
public ApiResponse<?> like(
        @PathVariable Long id,
        @RequestHeader(value = "X-Visitor-Id", required = false) String visitorId) {
    if (visitorId == null || visitorId.isEmpty()) {
        return ApiResponse.error(400, "visitorId不能为空");
    }
    // ...
}
```

**⚠️ 必须加 `required = false`**，否则不传 header 时 Spring 抛异常返回 500。

## 六、数据库表

### user_tokens

```sql
CREATE TABLE user_tokens (
    id          BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id     BIGINT NOT NULL,          -- 用户 ID
    token_hash  VARCHAR(128) NOT NULL,    -- SHA-256(token) 十六进制
    token_type  VARCHAR(10) NOT NULL,     -- 'ACCESS' | 'REFRESH'
    expires_at  DATETIME NOT NULL,        -- 过期时间
    created_at  DATETIME NOT NULL,
    revoked     TINYINT(1) DEFAULT 0,     -- 是否吊销
    replaced_by VARCHAR(128) DEFAULT NULL, -- Refresh Rotation 链
    UNIQUE KEY uk_token_hash (token_hash),
    INDEX idx_user_id (user_id)
);
```

### users 表相关字段

```sql
-- 密码存储格式（DelegatingPasswordEncoder）
password_hash VARCHAR(255)  -- 如 {noop}admin123 或 {bcrypt}$2a$...
role          VARCHAR(20)   -- admin / author / user
status        TINYINT       -- 1=正常 0=禁用
```

## 八、常见问题

### Q: 返回 401 但我觉得 Token 没问题？
检查 `user_tokens` 表: token 是否 `revoked=1`？`expires_at` 是否已过期？token_type 是否是 ACCESS？

### Q: 404 页面返回 HTML 不是 JSON？
当前 `add-mappings: false`，未匹配路径不会走到后端。如果配置了静态资源，SecurityConfig 需要排在前面的 `permitAll()`。

### Q: 密码怎么存？
使用 `DelegatingPasswordEncoder`，默认用 bcrypt。明文密码加 `{noop}` 前缀:
- 新密码自动 bcrypt 加密: `passwordEncoder.encode(plainText)`
- 旧明文兼容: 存 `{noop}admin123`
- 建议新注册用户都用 bcrypt

### Q: Controller 用 @Valid @RequestBody 校验，但返回 400 时说"服务器内部错误"?
检查 `GlobalExceptionHandler` 是否覆盖了对应异常。常见遗漏:
- `MissingServletRequestPartException`
- `MaxUploadSizeExceededException`
- `MethodArgumentTypeMismatchException`
