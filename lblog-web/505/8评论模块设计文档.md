# 评论模块设计文档

> 仅登录用户可评论，支持两层嵌套 + @提及
> 所有接口**已实现**，可直接对接前端

---

## 一、表结构

```sql
CREATE TABLE comments (
    id              BIGINT PRIMARY KEY AUTO_INCREMENT,
    post_id         BIGINT NOT NULL,              -- 所属文章

    -- 层级
    parent_id       BIGINT DEFAULT NULL,           -- NULL=顶级，非NULL=回复
    root_id         BIGINT DEFAULT NULL,           -- 所属顶级评论 ID（归组用）

    -- 作者
    user_id         BIGINT NOT NULL,               -- 登录用户 ID（仅登录用户可评论）
    author_name     VARCHAR(50) NOT NULL,           -- 冗余自 users.nickname
    author_avatar   VARCHAR(500) DEFAULT NULL,      -- 冗余自 users.avatar

    -- @ 提及
    reply_to_uid    BIGINT DEFAULT NULL,            -- 回复的目标用户 ID
    reply_to_name   VARCHAR(50) DEFAULT NULL,       -- 冗余显示名

    -- 内容 & 状态
    content         TEXT NOT NULL,
    status          TINYINT NOT NULL DEFAULT 0,     -- 0=待审 1=通过 2=驳回

    -- 计数（写操作维护，避免 COUNT 查询）
    like_count      INT NOT NULL DEFAULT 0,
    reply_count     INT NOT NULL DEFAULT 0,

    -- 审计
    ip_address      VARCHAR(45) DEFAULT NULL,
    created_at      DATETIME NOT NULL,
    updated_at      DATETIME NOT NULL,
    deleted_at      DATETIME DEFAULT NULL,
    is_delelte      INT DEFAULT 0,

    INDEX idx_post_status (post_id, status, created_at),
    INDEX idx_root (root_id),
    INDEX idx_parent (parent_id)
);
```

## 二、API 列表（全部已实现）

> 前台 base: `/iblogserver/api/v1`
> 创作中心 base: `/iblogserver/api/v1/author`
> 所有响应格式: `{ "code": 0, "message": "success", "data": ... }`

---

### 2.1 发表评论 `✅ 已实现`

```http
POST /api/v1/posts/{postId}/comments
Authorization: Bearer <token>
Content-Type: application/json

{
  "content": "写得很清楚！",
  "parentId": null
}
```

| 字段 | 必填 | 说明 |
|------|:----:|------|
| `content` | ✅ | 评论内容 |
| `parentId` | ❌ | `null`=顶级评论，有值=回复某条评论 |

**成功响应**:
```json
{ "code": 0, "message": "success", "data": { "id": 1 } }
```

**错误响应**:

| 场景 | code | message |
|------|:----:|---------|
| 未登录 | 401 | 请先登录 |
| 被回复的评论不存在 | 400 | 被回复的评论不存在 |

---

### 2.2 获取顶级评论 `✅ 已实现`

```http
GET /api/v1/posts/{postId}/comments?page=1&pageSize=10&sort=newest
```

| 参数 | 默认 | 说明 |
|------|:----:|------|
| `page` | 1 | 页码 |
| `pageSize` | 10 | 每页条数 |
| `sort` | newest | `newest`=最新 / `oldest`=最早 / `hot`=最热 |

**前端逻辑**: 子回复默认折叠，点击"查看 N 条回复"后调用接口 2.3

**成功响应**:
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "list": [
      {
        "id": 1,
        "author": { "id": 1, "nickname": "Ekko", "avatar": "/a.png" },
        "content": "写得很清楚！",
        "replyTo": null,
        "likeCount": 3,
        "replyCount": 2,
        "createdAt": "2026-05-05T10:00:00"
      }
    ],
    "total": 5,
    "page": 1,
    "pageSize": 10
  }
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `author.id` | number | 用户 ID |
| `author.nickname` | string | 显示名 |
| `author.avatar` | string/null | 头像 URL |
| `replyTo` | null | 顶级评论此字段为 null |
| `replyCount` | number | 该评论的回复总数，用于前端显示"查看 N 条回复" |
| `content` | string | 纯文本，前端自行处理 @ 高亮 |

### 2.3 获取子回复 `✅ 已实现`

```http
GET /api/v1/posts/{postId}/comments/{rootId}/replies?page=1&pageSize=10
```

| 参数 | 默认 | 说明 |
|------|:----:|------|
| `rootId` | — | 顶级评论 ID（路径参数） |
| `page` | 1 | |
| `pageSize` | 10 | |

**成功响应**:
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "list": [
      {
        "id": 2,
        "author": { "id": 2, "nickname": "Alice", "avatar": null },
        "content": "@Ekko 确实如此",
        "replyTo": { "id": 1, "nickname": "Ekko" },
        "likeCount": 1,
        "replyCount": 0,
        "createdAt": "2026-05-05T10:05:00"
      }
    ],
    "total": 2,
    "page": 1,
    "pageSize": 10
  }
}
```

**`replyTo` 字段说明**:
- `null` = 该评论是顶级评论
- 非 null = 该评论是回复，`replyTo.id` 是被回复人 ID，`replyTo.nickname` 是显示名
- 前端应显示为 `@{replyTo.nickname} 评论内容`，点击可跳转到被回复人

## 三、前端交互流程

```
文章详情页
│
├── 加载顶级评论 (接口 2.2)
│   └── 显示列表，每条显示 replyCount
│       └── 用户点击"查看 N 条回复"
│           └── 调用接口 2.3 加载子回复
│               └── 展开显示，支持分页
│
├── 发表评论 (接口 2.1)
│   ├── 输入框 → POST → 刷新列表
│   └── 回复某条评论 → 传 parentId → POST → 刷新回复
│
└── 创作中心 (接口 2.4-2.6)
    └── 评论管理 → 待审核列表 → 通过/驳回/删除
```

### 关键注意事项

1. **评论状态默认是待审核 (status=0)**
   - 用户发表后 **不会立即出现在前台**，需管理员审核通过
   - 可在评论区底部加提示：`"评论已提交，等待审核"`

2. **只有登录用户能评论**
   - 未登录用户看不到输入框
   - 发表接口返回 401 时弹出登录框

3. **回复时的 @ 由前端拼接**
   - 后端存的 `content` 不包含 `@xxx` 前缀
   - 前端拿到 `replyTo` 字段后，自己渲染 `@{replyTo.nickname}`
   - 建议：`<span class="at">@{replyTo.nickname}</span> {content}`

4. **回复数 (replyCount) 维护**
   - 新增回复时后端自动 +1，删除时自动 -1
   - 前端无需额外操作

5. **sort 参数**
   - `newest`: 最新优先（默认）
   - `oldest`: 最早优先
   - `hot`: 按 `like_count` 降序、时间降序

## 四、角色权限

| 操作 | 未登录 | user | author | admin |
|------|:-----:|:----:|:------:|:-----:|
| 查看评论 | ✅ | ✅ | ✅ | ✅ |
| 发表评论 | ❌ 401 | ✅ | ✅ | ✅ |
| 管理评论（审核/删除） | ❌ | ❌ | ✅ | ✅ |

---

## 五、审核流程

```
用户发表 → status=0(待审核)
    │
    ├── 管理员通过 → status=1 → 前台可见
    │
    └── 管理员驳回 → status=2 → 前台不可见
```
