# 图片上传 & 站点配置 — 后端接口设计

> 方案 B：Markdown 存相对路径，渲染时前端拼接 `imageBaseUrl`

---

## 接口 1：图片上传

```http
POST /api/v1/upload/image
```

### 请求头

| Header | 值 | 说明 |
|---|---|---|
| `Authorization` | `Bearer <token>` | 需要 JWT，仅作者/管理员可用 |
| `Content-Type` | `multipart/form-data` | |

### 请求参数（FormData）

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `file` | File | 是 | 图片文件 |

### 限制

| 限制项 | 值 |
|---|---|
| 文件大小 | ≤ 10 MB |
| 支持格式 | `jpg`, `png`, `gif`, `webp`, `svg` |

### 成功响应

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "url": "/uploads/2025/05/a1b2c3d4-e5f6-7890.jpg",
    "filename": "a1b2c3d4-e5f6-7890.jpg",
    "size": 234567,
    "mimeType": "image/jpeg"
  }
}
```

### 错误响应

**格式不支持（400）**

```json
{
  "code": 400,
  "message": "不支持的图片格式，仅支持 jpg/png/gif/webp/svg",
  "data": null
}
```

**超出大小限制（413）**

```json
{
  "code": 413,
  "message": "图片大小超过限制（最大 10MB）",
  "data": null
}
```

**未登录/无权限（401）**

```json
{
  "code": 401,
  "message": "未登录或 Token 已过期",
  "data": null
}
```

### 关键约定

1. **`url` 返回相对路径**（以 `/` 开头），不包含域名。后续切换 OSS 时已发布文章无需改动
2. **路径格式**：`/uploads/{yyyy}/{mm}/{uuid}.{ext}`
3. **UUID 命名**：后端生成，避免文件名冲突
4. **响应中的 `url` 即 Markdown 中存储的内容**，例如：`![](/uploads/2025/05/a1b2c3d4-e5f6-7890.jpg)`

## 接口 2：获取站点配置

```http
GET /api/v1/config
```

### 请求头

无需登录，公开接口。

### 成功响应

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "imageBaseUrl": "https://oss-cn.com/images",
    "imageMaxSize": 10485760
  }
}
```

### 字段说明

| 字段 | 类型 | 说明 |
|---|---|---|
| `imageBaseUrl` | string | 图片基础 URL。Markdown 渲染时，前端对以 `/` 开头的图片 src 拼接此前缀。**开发环境可返回空字符串**，此时走 Vite proxy |
| `imageMaxSize` | number | 图片大小上限（字节）。前端上传前做本地校验，减少无效的失败请求 |

### 扩展性

后续如需新增站点级的配置项（站点名称、SEO 描述、评论开关等），直接在本接口追加字段即可，无需新增接口。

## 前后端联调用例

```
① 用户粘贴/选择图片
    ↓
② POST /api/v1/upload/image → 后端返回 {"url": "/uploads/2025/05/uuid.jpg"}
    ↓
③ 前端将 ![](/uploads/2025/05/uuid.jpg) 插入 Markdown 编辑器光标位置
    ↓
④ 渲染时 MarkdownRenderer:
   - src 以 "/" 开头 → 拼接 imageBaseUrl
   - 最终 <img src="https://oss-cn.com/images/uploads/2025/05/uuid.jpg">
    ↓
⑤ 切换 OSS 只需改 imageBaseUrl，已发布文章不受影响
```

## 前端计划改动范围

| 文件 | 改动内容 |
|---|---|
| `src/services/api.ts` | 新增 `uploadImage(file)` 和 `getSiteConfig()` 方法 |
| `src/components/MarkdownRenderer.tsx` | 新增 `imageBaseUrl` prop，拦截 `<img>` 拼接相对路径 |
| `src/pages/author/PostEditor.tsx` | 粘贴/选择图片 → 上传 → 在光标位置插入 Markdown |
| `src/contexts/SiteDataContext.tsx` | 从 `getSiteConfig()` 获取并共享 `imageBaseUrl` |
